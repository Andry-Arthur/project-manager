# Supabase Integration Guide for MadaProject

This guide explains how to configure MadaProject to use Supabase for database, authentication, and storage.

## Prerequisites

- A Supabase account (sign up at [supabase.com](https://supabase.com))
- Node.js 18+ installed
- npm or pnpm package manager

## Step 1: Create a Supabase Project

1. Go to [supabase.com](https://supabase.com) and sign in
2. Click "New Project"
3. Fill in project details:
   - **Name**: MadaProject (or your preferred name)
   - **Database Password**: Choose a strong password (save it securely)
   - **Region**: Choose the closest region to Madagascar (likely Africa or Europe)
4. Click "Create new project" and wait for provisioning (2-5 minutes)

## Step 2: Get Your Supabase Credentials

Once your project is ready:

1. Go to **Project Settings** (gear icon in sidebar)
2. Navigate to **API** section
3. Copy the following values:
   - **Project URL**: `https://xxxxx.supabase.co`
   - **anon/public key**: `eyJhbGc...` (long string)
   - **service_role key**: `eyJhbGc...` (keep this secret!)

## Step 3: Configure Environment Variables

1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```

2. Edit `.env` and add your Supabase credentials:
   ```env
   # Supabase Configuration
   SUPABASE_URL="https://your-project.supabase.co"
   SUPABASE_ANON_KEY="your-anon-key-here"
   SUPABASE_SERVICE_KEY="your-service-role-key-here"

   # Database Connection String
   # Get this from: Project Settings → Database → Connection string → URI
   # Use Transaction mode (port 6543) for Prisma
   DATABASE_URL="postgresql://postgres.[project-ref]:[your-password]@aws-0-[region].pooler.supabase.com:6543/postgres"
   ```

### Getting the DATABASE_URL

1. In Supabase dashboard, go to **Project Settings** → **Database**
2. Scroll to **Connection string** section
3. Select **URI** tab
4. Choose **Transaction** mode (port 6543)
5. Copy the connection string and replace `[YOUR-PASSWORD]` with your actual database password

## Step 4: Install Dependencies

```bash
npm install
```

This will install:
- `@supabase/supabase-js` - Supabase JavaScript client
- All other project dependencies

## Step 5: Set Up Database Schema

### Option A: Using Prisma Migrations (Recommended)

1. Generate Prisma client:
   ```bash
   npm run db:migrate
   ```

2. This will create all necessary tables in your Supabase database based on the schema in `packages/db/prisma/schema.prisma`

### Option B: Using Supabase SQL Editor

If you prefer to use Supabase's SQL editor:

1. Go to **SQL Editor** in Supabase dashboard
2. Run the migration SQL (generated by Prisma)
3. Or manually create tables based on the schema

## Step 6: Configure Supabase Storage

Create storage buckets for file uploads:

1. Go to **Storage** in Supabase dashboard
2. Create the following buckets:
   - `project-attachments` (for task/project attachments)
   - `user-avatars` (for user profile pictures)
   - `company-logos` (for company logos)
   - `project-documents` (for project documentation)
   - `project-exports` (for exported reports)

3. Set bucket policies:
   - **project-attachments**: Private (authenticated users only)
   - **user-avatars**: Public (for profile pictures)
   - **company-logos**: Public (for company branding)
   - **project-documents**: Private (authenticated users only)
   - **project-exports**: Private (authenticated users only)

### Example Bucket Policy for Private Access

```sql
-- Allow authenticated users to upload to project-attachments
CREATE POLICY "Authenticated users can upload files"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (bucket_id = 'project-attachments');

-- Allow users to read their own files
CREATE POLICY "Users can read own files"
ON storage.objects FOR SELECT
TO authenticated
USING (bucket_id = 'project-attachments');
```

## Step 7: Configure Supabase Authentication

1. Go to **Authentication** → **Providers** in Supabase dashboard
2. Enable **Email** provider (enabled by default)
3. Configure additional providers if needed:
   - Google OAuth
   - GitHub OAuth
   - etc.

### Email Configuration

For production email:

1. Go to **Project Settings** → **Auth** → **Email Templates**
2. Configure your SMTP settings (or use Supabase's default)
3. Customize email templates for:
   - Confirmation emails
   - Password recovery
   - Magic link emails

## Troubleshooting

### Connection Issues

If you get connection errors:

1. **Check DATABASE_URL format**:
   - Must use port 6543 (Transaction mode)
   - Password must be URL-encoded if it contains special characters

2. **Check Supabase project status**:
   - Go to Supabase dashboard → Project Settings
   - Ensure project is active

3. **Check firewall/network**:
   - Ensure your network allows connections to Supabase

### Authentication Issues

If authentication fails:

1. **Verify JWT_SECRET** in `.env` matches your Supabase JWT secret
2. **Check user exists** in both Supabase Auth and your database
3. **Clear browser cache** and try again

### Prisma Migration Errors

If migrations fail:

1. **Reset database** (warning: deletes all data):
   ```bash
   npx prisma migrate reset --schema=packages/db/prisma/schema.prisma
   ```

2. **Check migration status**:
   ```bash
   npx prisma migrate status --schema=packages/db/prisma/schema.prisma
   ```

## Production Deployment

### Environment Variables for Production

Ensure these are set in your production environment:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_KEY=your-service-role-key
DATABASE_URL=postgresql://...
JWT_SECRET=your-production-jwt-secret
NODE_ENV=production
```

### Security Considerations

1. **Never expose service_role key** to clients
2. **Use Row Level Security (RLS)** in Supabase for additional security
3. **Enable 2FA** for Supabase account
4. **Set up proper CORS** in Supabase dashboard
5. **Use environment-specific** Supabase projects (dev, staging, prod)

### Performance Optimization

1. **Enable Connection Pooling**:
   - Use Supabase's built-in connection pooling (port 6543)
   - Configure Prisma for optimal pool size

2. **Set up Caching**:
   - Use React Query caching on frontend
   - Consider Redis for API caching

3. **Monitor Performance**:
   - Use Supabase's built-in monitoring
   - Set up alerts for slow queries
